HTML Structure

<!DOCTYPE html>

Declares the document type and version of HTML (HTML5 in
this case).

<html lang="en">

Defines the root element of the HTML document and sets the language
to English.

<head>

Contains meta-information about the document, such as font imports, character set, and links to stylesheets, which are not directly visible to users.

<style>

A block where internal CSS is defined, controlling the appearance of the
web page.

<body>

Contains the visible content of the web page, including text, images, links, and any interactive elements.

<div>

A block-level container element used to group and structure content within the
<body>.

<p>

Defines paragraphs in the HTML structure, used for block-level
text content.

<h1> to <h7>

Heading elements define different levels of headings, with <h1> being the most important and <h7> means the max amount of heading you can add.

Typography and Fonts

@font-face

CSS rule used to load custom fonts from a source, such as Montserrat in
different weights.

font-family

Specifies the font or a stack of fonts for text. For instance, 'Montserrat', Helvetica, sans-serif.

font-weight

Controls the thickness of text. E.g., 300 (light), 500 (medium), and
600 (bold).

font-size

Sets the size of the text, commonly expressed in viewport height (vh).

letter-spacing

Controls the space between characters, affecting the overall visual appeal of
the text.

line-height

Defines the space between lines of text in an element.

-webkit-text-stroke

Adds a stroke (outline) to text. The stroke color and width are defined.

Positioning and Layout

position: absolute;

Positions the element relative to its closest positioned ancestor or the initial containing block.

top, left, right, bottom

Defines the position of an absolutely positioned element.

width, height

Specifies the dimensions of an element, commonly expressed in vw
and vh.

margin

Defines the space outside an element.

padding

Defines the space inside an element, between the content and
its border.


Grid Layout
display: grid;

Defines a grid-based layout system, allowing the placement of elements in rows and columns.

grid-template-columns

Defines the number and width of columns in the grid.
Example: repeat(5, 15vw);.

grid-template-rows

Defines the number and height of rows in the grid.
Example: repeat(3, 21vh);.

gap

Specifies the spacing between rows and columns in a grid layout.

Box and Div Styling

.rectangle

Defines a box-like element with properties like height, width, border, and background color.

border

Defines the border around an element. Example: border: 0.3vh solid #161515;.

box-shadow

Adds shadow effects to elements, giving the element a subtle depth.

text-align

Aligns text horizontally within an element. Example: text-align: left;.


Images and Media
<img>

Embeds images into the HTML document. The src attribute specifies the
image source.

max-width, height: auto;

Ensures that images resize responsively without distorting their ratio.

<iframe>

Embeds another HTML document within the current page using the
src attribute.

Basic Shapes

.rectangle1, .rectangle2, .rectangle3

Defines various rectangular shapes with consistent borders
and shadows.

border-radius

Rounds the corners of a box element. Example: border-radius: 10px;.

box-shadow: inset

Creates a shadow inside the element’s frame, giving a 3D or
inset effect.


Color and Background
background-color

Defines the background color of elements. Example: background-color: #f2f2f2;.

color

Sets the text color for an element. Example: color: #161515; applies a dark shade to the text.

background-size: absolute;

Ensures that the background image or color is fixed and does not repeat or scale.

Variables and Constants

let

Declares block-scoped local variables, optionally initializing them to
a value.

const

Declares block-scoped constants that cannot be reassigned after
the initialization.

mouseX, mouseY

Returns the current horizontal (mouseX) and vertical (mouseY) positions of the mouse.

width, height

Returns the width and height of the canvas.

PI, HALF_PI

Mathematical constants for π and π/2 respectively.

Functions in p5.js

preload()

Loads external assets (like images, sounds) before the program starts.

setup()

Initializes variables and settings. It runs once at the start.

draw()

Continuously executes the lines of code inside its block until the
program stops.

mousePressed(), mouseDragged(), mouseReleased()

Detects when the mouse button is pressed, dragged, or released.

keyTyped()

Detects when a key is typed.

clear()

Clears the canvas, removing everything drawn previously.

Graphics and Drawing

createCanvas(), createGraphics()

Sets the size of the canvas and creates off-screen graphics buffers.

ellipse(), line(), rect(), arc(), circle()

Draws shapes (ellipse, line, rectangle, arc, circle) on the canvas.

background(), fill(), noFill()

Sets the background color, fill color for shapes, or disables
shape filling.

noStroke(), stroke(), strokeWeight()

Disables or enables stroke (border) with color and thickness settings.

text(), textSize(), textAlign()

Draws text on the canvas and sets its size and alignment.

push(), pop()

Saves and restores the current drawing style settings and transformations. Used to code something that do not follow the general rules of the rest.

map(), dist()

Maps a number from one range to another and calculates the distance between two points.

Fonts and Text

loadFont()

Loads a custom font for displaying text.

textToPoints(), textBounds()

Converts text into an array of vector points and calculates its
bounding box.


Physics and Motion
angle, aVelocity, aAcceleration

Custom variables representing angular motion.

sin(), cos(), atan2()

Trigonometric functions used to calculate angles and positions.

Arrays

push(), pop()

Adds an element to the end of an array or removes the last element.


Audio in p5.js
loadSound()

Loads sound files for playback.

song.play(), song.stop(), song.loop()

Controls playback of a sound file.

amp(), rate()

Sets the volume and playback speed for a sound.

getAudioContext()

Retrieves the audio context to handle audio input/output in the browser.


Color and Background
colorMode()

Sets the color mode (RGB or HSB) for colors.

background(), fill(), noStroke()

Sets the background color of the canvas and controls fill and
stroke settings.

Mouse and Keyboard Interaction

mouseX, mouseY

Returns the current horizontal and vertical position of the mouse.

mousePressed(), mouseDragged(), mouseReleased()

Detects when the mouse is pressed, dragged, or released.

keyPressed(), keyTyped()

Detects when a key is pressed or typed.


Image Manipulation and Brush Layering
loadImage()

Loads an image file into the program.

image()

Displays an image on the canvas.

createGraphics()

Creates an off-screen graphics buffer for more complex visual effects.

erase(), p.circle()

Erases content from the graphics buffer and draws a circle on the off-screen layer.

Techniques

Don't treat AI like human

While AI may sometimes appear limited in its capabilities, especially as you gain more experience using it, it can still be an invaluable tool for coding, particularly with complex tasks. Even if you're proficient in coding, leveraging AI to generate an initial draft and refining it afterward can significantly reduce development time. However, it's crucial that your commands are precise and tailored to how AI processes information. Be specific about parameters such as margin widths, text sizes, and other details. For more challenging code, while you don't need to know every detail of implementation, having a general idea of the commands required is essential. If AI provides unexpected output, verify the functionality to ensure it aligns with your goals. Ultimately, AI functions as a predictive system, so the more detailed and clear your instructions, the more accurate the results will be. Think of it as guiding a novice—without clear direction, the outcome may not meet expectations.

Keep everything easy

As a beginner coder, it's important not to overcomplicate things. What seems basic can often be challenging to code. It's better to focus on developing ideas rather than getting caught up in complex coding. For example, I once had an ambitious idea involving three scenes and 12 monitors—impressive, but beyond my capabilities and unsuitable for the course. When tackling coding tasks, aim for what seems simple, as you'll likely still face challenges even at that level.

Use Illustrator as a prototyping tool

When designing a website, it's helpful to use Illustrator to quickly develop a brief layout. It's much faster than moving elements around in HTML or JavaScript. The basic tools in Illustrator can replicate most elements you would create in HTML, allowing you to use it as a design space. Once everything is set, you can then move to coding.

Know when to use Html when to use p5

As a designer, it's crucial to know which coding language to use for different tasks, as this will save time and improve your results. Think of p5.js as the Photoshop or Illustrator of the coding world—it's where you create complex animations and dynamic visuals. HTML, on the other hand, is like InDesign, where you organise and structure everything. A website is essentially a digital publication, so HTML is the primary tool for building it, with p5.js used to assist in creating more
complex effects.

Build a huge framework before getting into detail

Don't spend too much time on small details early in the process, as you may later realise you didn't need them after all. Focus on the framework first—your content, the overall layout, and key elements like the movement of kinetic text. Details such as font size, letter spacing, or minor positioning issues can be addressed later. For example, I had issues with a misaligned heading and a font not loading, but I chose to finish writing the content and placing images before circling back to fix them. Remember, we have deadlines. Having a solid framework to submit is better than getting stuck on details and feeling stressed close to the deadline with nothing substantial to show. In addition, if you find a bug, go to web inspector to see what is going wrong.

Write descrption near your code

Code can get very long, and it's easy to get lost when it reaches 400 or 500 lines. Adding comments as you go can help you quickly find and fix specific sections when issues arise. In HTML and p5.js, use //Your text to comment you code. If you have further questions, you can visit resources like W3Schools or the p5.js reference guide.
